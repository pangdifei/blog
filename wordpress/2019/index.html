内容管理系统CMS生成静态页面

Posted on 三月 13, 2019 by pdf

静态网站在国内是高端标配例如拓尔思TRS的WCM，低配的有开源的织梦dedecms或者phpcms都属于半静态页面生成方式。这类开源项目多存在安全漏洞而且处理缓慢，要想实际使用需要大量改造。

WordPress ，Hugo，Jekyll 成为一种适当的选择。其中Wordpress更是用量第一开源项目，你需要安装wp2static插件，就可以完成静态化处理。静态页面上传到任何你希望的地方，wordpress只需要部署在你的内网甚至笔记本里，对外人它存在。这是静态网站的优点不必多说。

剩下的问题是改造一个合适的wordpress模板和添加内容。pinegrow和
Pinegrow WordPress Starter Theme 2 https://github.com/Pinegrow/PGWPBootstrap4StarterTheme是我当前的选择。

github page看着域名加后缀不爽https://pangdifei.github.io/blog ,就试试
GITHUB ＋ NETLIFY 例如我的https://pangdifei.netlify.com/

其他静态生成程序 Hugo 需要在命令行操作，不太适于非技术人员。如果markdown能解决直接粘贴图片。估计还需要时间 ，typora是个快速接近目标的markdown编辑器。其他静态程序没用过给他们加油吧，https://www.staticgen.com/
